11. Explain general two dimensional pivot point rotation and derive the composite matrix. (8)

Answer

Composite transformations are sequences of transformations such as translations, rotations, and scalings applied to objects to achieve the desired effect.

Composite transformations involve combining multiple basic transformations into a single transformation matrix, which can then be applied to an object.

These transformations are represented by matrices in 3D space and are typically 4x4 matrices to accommodate homogeneous coordinates.

Example

Suppose we want to scale an object, then rotate it around the Y-axis, and finally translate it to a new position.

  1. Scale the object by a factor of 2.
  1. Rotate the object by 45 degrees around the Y-axis.
  1. Translate the object to the position (5, 3, -2).
#include <GL/glut.h>

void display() {
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
    glLoadIdentity();

    // Composite Transformation Matrix
    glPushMatrix();

    // Step 3: Translation
    glTranslatef(5.0f, 3.0f, -2.0f);

    // Step 2: Rotation around the Y-axis
    glRotatef(45.0f, 0.0f, 1.0f, 0.0f);

    // Step 1: Scaling
    glScalef(2.0f, 2.0f, 2.0f);

    // Draw a cube (for example)
    glutWireCube(1.0);

    glPopMatrix();

    glutSwapBuffers();
}

int main(int argc, char** argv) {
    glutInit(&argc, argv);
    glutInitDisplayMode(GLUT_DOUBLE | GLUT_RGB | GLUT_DEPTH);
    glutCreateWindow("Composite 3D Transformations");
    glutDisplayFunc(display);
    glEnable(GL_DEPTH_TEST);
    glutMainLoop();
    return 0;
}

17. Explain the basic principle of OpenGL geometric transformations. (6)

Answer

Translation

  • Function: glTranslate* (tx, ty, tz);
  • Purpose: Translates an object by specified distances along the x, y, and z axes.
  • Parameters:
    • tx: Translation along the x-axis.
    • ty: Translation along the y-axis.
    • tz: Translation along the z-axis (set to 0.0 for 2D transformations).
  • Example:
    • To translate an object 25 units along the x-axis and -10 units along the y-axis:
    glTranslatef (25.0, -10.0, 0.0);
  • Effect: Objects defined after this function call are translated according to the specified parameters.

Rotation

  • Function: glRotate* (theta, vx, vy, vz);
  • Purpose: Rotates an object around a specified axis.
  • Parameters:
    • theta: Rotation angle in degrees.
    • vx, vy, vz: Components of the vector that defines the rotation axis.
  • Example:
    • To rotate an object 90° around the z-axis:
    glRotatef (90.0, 0.0, 0.0, 1.0);
  • Effect: Rotates objects defined after this function call. In 2D systems, rotation is typically around the z-axis.

Scaling

  • Function: glScale* (sx, sy, sz);
  • Purpose: Scales an object by specified factors along the x, y, and z axes.
  • Parameters:
    • sx: Scaling factor along the x-axis.
    • sy: Scaling factor along the y-axis.
    • sz: Scaling factor along the z-axis (set to 1.0 for 2D transformations).
  • Example:
    • To scale an object by 2 in the x direction, 3 in the y direction, and reflect it across the x-axis:
    glScalef (2.0, -3.0, 1.0);
  • Effect: The specified scaling and reflection transformations are applied to objects defined after this function call.

18. Explain the following open GL geometric transformation functions
a.
glMatrixMode().
b.
glPopMatrix. (8)

Answer

glMatrixMode

  • Specifies current matrix for geometric-viewing transformations, projection transformations, texture transformations, or color transformations.
  • The default argument for the glMatrixMode function is GL_MODELVIEW, which designates the 4×4 modelview matrix as the current matrix.
// Set the current matrix mode to the modelview matrix
glMatrixMode(GL_MODELVIEW);

// Set the current matrix mode to the projection matrix
glMatrixMode(GL_PROJECTION);

// Set the current matrix mode to the texture matrix
glMatrixMode(GL_TEXTURE);

OpenGL Matrix Stacks

We have two functions available in OpenGL for processing the matrices in a stack. These stack-processing functions are more efficient than manipulating the stack matrices individually, particularly when the stack functions are implemented in hardware.

glPushMatrix()

Copies the top matrix in the stack and store copy in the second stack position.

// Push the current matrix onto the stack
glPushMatrix();

This gives us duplicate matrices at the top two positions of the stack.

glPushMatrix() is called to save the current state of the matrix. This is useful when you want to apply transformations to only certain objects without affecting others.

glPopMatrix()

Erases the top matrix in the stack and moves the second matrix to the top of the stack.

// Pop the current matrix from the stack
glPopMatrix();

Example

glMatrixMode (GL_MODELVIEW);

glColor3f (0.0, 0.0, 1.0); // Set current color to blue.
glRecti (50, 100, 200, 150); // Display blue rectangle.

glPushMatrix ( ); // Make copy of identity (top) matrix.
glColor3f (1.0, 0.0, 0.0); // Set current color to red.

glTranslatef (-200.0, -50.0, 0.0); // Set translation parameters.
glRecti (50, 100, 200, 150); // Display red, translated rectangle.

glPopMatrix ( ); // Throw away the translation matrix.
glPushMatrix ( ); // Make copy of identity (top) matrix.

glRotatef (90.0, 0.0, 0.0, 1.0); // Set 90-deg. rotation about z axis.
glRecti (50, 100, 200, 150); // Display red, rotated rectangle.

glPopMatrix ( ); // Throw away the rotation matrix.
glScalef (-0.5, 1.0, 1.0); // Set scale-reflection parameters.
glRecti (50, 100, 200, 150); // Display red, transformed rectangle.

21. Explain OpenGL raster transformations and OpenGL geometric transformation functions. (6)

Answer

OpenGL Raster Operations

Raster operations involve manipulating arrays of pixel values, which are stored in buffers. These operations can include retrieving pixel values, copying pixel blocks from one buffer to another, or performing logical operations on pixel arrays.

glReadPixels

  • Used to select and retrieve a rectangular block of pixel values from a designated buffer.
  • Syntax:
glReadPixels (xmin, ymin, width, height, dataFormat, dataType, array);

Parameters:

  • xmin, ymin: Coordinates of the lower-left corner of the rectangular block.
  • width, height: Size of the block in terms of pixels.
  • dataFormat: Specifies the format of the pixel data (e.g., GL_DEPTH_COMPONENT for depth buffer, GL_STENCIL_INDEX for stencil buffer).
  • dataType: Specifies the data type of the pixel data.
  • array: Pointer to the array where the pixel data will be stored.

glReadBuffer

  • Selects a specific buffer from which pixel data will be read.
  • Syntax:
glReadBuffer (buffer);

glCopyPixels

  • Copies a block of pixel data from one location to another within the same set of OpenGL buffers.
  • Syntax:
glCopyPixels (xmin, ymin, width, height, pixelValues);

glEnable(GL_COLOR_LOGIC_OP) and glLogicOp(logicOp)

  • Enable and specify a bitwise logical operation for combining incoming and destination pixel color values.
  • Logical operations: GL_AND, GL_OR, GL_XOR, etc.
  • Invert operations: GL_COPY_INVERTED, GL_INVERT.

Advanced Raster Operations

glPixelTransfer, glPixelMap

  • These functions can shift or adjust pixel values, including color, depth, or stencil values.
